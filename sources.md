# Sources and basis

This public proof surface is based on general agent workflow lessons:

- state sources should be explicit and fresh;
- public, paid, legal, wallet, destructive, and real-data actions need clear gates;
- completion claims should be evidence-backed;
- reviewer feedback should become tests or checklist items;
- repeated zero-outcome routes should be killed or revised.

It does not quote or expose private customer material, private logs, private task boards, secrets, or real client data.
